# calamine-js

Fast xlsx parsing for JavaScript, built on the Rust
[calamine](https://crates.io/crates/calamine) crate. Monorepo for:

| Package | What it is |
|---|---|
| [`@krllc/calamine-wasm`](./packages/calamine-wasm) | WASM build + streaming, worker-first TypeScript client for Node.js and browsers. **The primary package.** |
| [`@krllc/calamine-native`](./packages/calamine-native) | NAPI addon (path-based, fastest, Node only). Optional engine for the wasm package's client (`engine: "native"`). Current-platform builds only for now — no prebuild matrix yet. |

Rust layout (cargo workspace):

- `crates/core` — everything binding-agnostic: JSON serialization (incl. the
  `{"d": serial}` date tagging the client decodes), the resumable pull cursor,
  and the workbook state machine. Exists so the two engines produce
  byte-identical output by construction.
- `crates/wasm`, `crates/native` — thin wasm-bindgen / napi shims over core.

Design rationale and measurements: [packages/calamine-wasm/DESIGN.md](./packages/calamine-wasm/DESIGN.md).

## Building

Requires Rust (`rustup target add wasm32-unknown-unknown` for wasm), Node 20+.

```sh
npm install
npm run build      # native addon (current platform) + wasm (node/web targets) + TS client
npm test           # Node client tests + real-browser tests (headless Chromium)
```

Test fixture `packages/calamine-wasm/test/fixtures/sms-small.xlsx` is checked
in (deterministic, generated by the companion benchmark harness).

## Benchmarks

The comparative benchmark harness (vs SheetJS, 1 MB–100 MB corpora) lives in a
